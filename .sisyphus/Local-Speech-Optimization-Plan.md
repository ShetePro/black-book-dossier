# Black Book - 本地语音识别优化计划

## 目标
在不使用云端 API 的前提下，最大化本地语音识别的准确率。

---

## 当前方案分析

### 现有实现
- **模型**: Whisper RN (OpenAI Whisper 本地移植)
- **模型大小**: ~150MB (tiny 或 base)
- **准确率**: 约 80% (中文)
- **延迟**: 2-5 秒

### 瓶颈分析
1. **模型太小** - tiny/base 模型容量有限
2. **缺少语言模型** - 没有中文语言模型后处理
3. **音频质量** - 缺少降噪和语音活动检测(VAD)
4. **领域不适配** - 模型未针对商务/人名场景优化

---

## 优化方案

### Phase 1: 模型升级（最重要）

#### 方案 A: 升级 Whisper 模型大小

**当前**: Whisper Tiny (~39MB) 或 Base (~74MB)
**升级**: Whisper Small (~244MB) 或 Medium (~769MB)

**效果预估**:
- Tiny -> Small: +10-15% 准确率
- Small -> Medium: +5-10% 准确率

**代价**:
- 内存占用增加 3-5 倍
- 首次加载时间增加 3-5 秒
- 识别延迟增加 1-2 秒

**实施步骤**:
1. [ ] 下载 Whisper Small 模型
2. [ ] 适配 whisper.rn 库使用 Small 模型
3. [ ] 测试性能和准确率
4. [ ] 权衡后决定是否升级到 Medium

#### 方案 B: 使用 Faster-Whisper (推荐)

**技术**: CTranslate2 优化的 Whisper
**优势**:
- 相同准确率下速度快 2-3 倍
- 内存占用减少 50%
- 支持量化 (INT8) 进一步减少内存

**模型大小对比**:
| 模型 | 原始 Whisper | Faster-Whisper (FP16) | Faster-Whisper (INT8) |
|------|-------------|----------------------|---------------------|
| Tiny | 39MB | 39MB | 20MB |
| Base | 74MB | 74MB | 37MB |
| Small | 244MB | 244MB | 122MB |
| Medium | 769MB | 769MB | 385MB |

**实施步骤**:
1. [ ] 研究 faster-whisper 在 React Native 的可行性
2. [ ] 寻找或创建 Native Module 绑定
3. [ ] 替换现有 whisper.rn

#### 方案 C: 使用中文专用模型 (最佳效果)

**候选模型**:

1. **Paraformer-zh** (阿里达摩院)
   - 大小: ~200MB
   - 中文准确率: 90%+
   - 专门为中文优化

2. **WeNet** (出门问问)
   - 大小: ~150MB
   - 支持流式识别
   - 中文场景优秀

3. **SenseVoice** (阿里)
   - 大小: ~230MB
   - 多语言支持好
   - 中文准确率高

**推荐**: Paraformer-zh 或 WeNet

**实施步骤**:
1. [ ] 调研 React Native 绑定方案
2. [ ] 集成 ONNX Runtime 或 MNN
3. [ ] 加载并运行模型

---

### Phase 2: 音频预处理优化

#### 任务 1: 语音活动检测 (VAD)

**作用**: 自动识别有效语音片段，去除静音和噪声

**效果**:
- 减少误识别
- 提升用户体验（自动停止录音）

**技术选型**:
- **Silero VAD** - 轻量级，准确率高，支持移动端
- **WebRTC VAD** - Google 出品，实时性好

**实施步骤**:
1. [ ] 集成 Silero VAD 模型 (~1MB)
2. [ ] 实时检测语音活动
3. [ ] 自动裁剪有效语音段
4. [ ] 自动停止录音（检测到 1-2 秒静音）

#### 任务 2: 音频降噪

**作用**: 去除背景噪声，提升信噪比

**技术**:
- **RNNoise** - 轻量级实时降噪
- **SpeexDSP** - 经典降噪算法

**效果**: +3-5% 准确率提升

**实施步骤**:
1. [ ] 集成 RNNoise (~50KB)
2. [ ] 录音前进行降噪处理
3. [ ] 保存降噪后的音频用于识别

#### 任务 3: 音频增强

**技术**:
- 音量归一化
- 预加重滤波
- 高频增强

**实施**: 使用 Web Audio API 或原生音频处理

---

### Phase 3: 后处理优化

#### 任务 4: 添加语言模型 (LM)

**作用**: 利用上下文纠正识别错误

**技术**:
- **KenLM** - 轻量级 N-gram 语言模型
- **训练语料**: 商务对话、联系人姓名

**效果**: +5-10% 准确率提升

**实施步骤**:
1. [ ] 收集商务对话语料
2. [ ] 训练 3-gram 语言模型 (~10MB)
3. [ ] 集成 KenLM 到识别流程
4. [ ] 对 Whisper 输出进行重打分

#### 任务 5: 自定义热词/人名识别

**作用**: 提升特定人名、公司名的识别准确率

**实现方式**:
1. **提示词 (Prompt)**
   - Whisper 支持 prompt，可以提示常见人名
   - 例如: "以下是商务对话，涉及以下人物: 张三、李四..."

2. **后处理替换**
   - 建立联系人姓名词典
   - 识别后模糊匹配替换

**实施步骤**:
1. [ ] 从联系人中提取所有人名
2. [ ] 构建拼写检查词典
3. [ ] 识别结果模糊匹配替换
4. [ ] 更新 Whisper prompt

---

### Phase 4: 架构优化

#### 任务 6: 模型量化

**技术**: INT8 量化
**效果**: 
- 模型大小减少 50%
- 内存占用减少 50%
- 速度提升 1.5-2 倍
- 准确率损失 < 2%

**实施**: 使用 ONNX Runtime 的量化工具

#### 任务 7: 流式识别

**当前**: 整段录音后识别（延迟高）
**优化**: 边录音边识别（实时显示）

**技术**:
- Whisper 支持 chunk-based 识别
- 或使用 SenseVoice/WeNet 的流式模式

**效果**: 用户体验大幅提升

## 推荐实施路径

### 路径 A: 快速优化（1-2 天）
适合：时间紧，想快速看到效果

1. **升级 Whisper 模型到 Small** (+10-15%)
2. **添加 VAD 自动停止** (+3-5%)
3. **使用 Prompt 提示人名** (+3-5%)

**预期准确率**: 80% → **90-95%**

### 路径 B: 深度优化（1 周）
适合：追求极致本地识别效果

1. **切换到 Paraformer-zh** (+15-20%)
2. **添加 VAD + 降噪** (+5-8%)
3. **集成语言模型** (+5-10%)
4. **流式识别优化体验**

**预期准确率**: 80% → **93-98%**

### 路径 C: 混合方案（推荐）
适合：平衡效果和隐私

1. **本地优化到 90% 准确率**（路径 A）
2. **网络好时上传到云端二次校正**（可选）
3. **用户可选择纯本地模式**
