# 修复拼音匹配问题 - 施迦奇 vs 张三

## 问题描述
语音识别将"张三"误识别为"施迦奇"，但系统没有正确匹配到联系人。

## 根本原因
- "施迦奇" (shī jiā qí) 和 "张三" (shī jiā qí) 的拼音完全相同
- 但 `services/transcription/similarity.ts` 中的 `charToPinyin` 映射表缺少：
  - "迦" → "jia"
  - "祺" → "qi"

没有这些映射时，拼音相似度计算会失败，导致无法匹配同音字。

## 修复方案

### 文件修改
**文件**: `services/transcription/similarity.ts`

在 `charToPinyin` 映射表中添加缺失的字符：

```typescript
// 在现有映射后添加以下字符

// 修复：施迦奇 -> 张三 的匹配问题
'迦': 'jia',
'祺': 'qi',

// 补充其他常见同音字（预防类似问题）
'伽': 'jia', '珈': 'jia',
'琪': 'qi', '琦': 'qi', '绮': 'qi',
```

### 需要检查的字符
在添加前，先检查以下字符是否已存在于映射表中：
- ✅ '施' - 已存在 (shi)
- ✅ '佳' - 已存在 (jia)  
- ❓ '迦' - 需要添加
- ❓ '祺' - 需要添加
- ❓ '琪' - 需要添加
- ❓ '琦' - 需要添加

## 验证步骤
1. 修改 `similarity.ts` 添加缺失映射
2. 运行 `npx tsc --noEmit` 确保无类型错误
3. 测试用例：
   - 输入: "施迦奇今天写了一中午的代码"
   - 预期: 匹配到联系人 "张三"
   - 相似度: > 0.9 (拼音完全相同)

## 影响范围
- 仅影响 `services/transcription/similarity.ts`
- 增强拼音相似度计算，帮助识别同音字错误
- 向后兼容，只添加新映射不修改现有逻辑
